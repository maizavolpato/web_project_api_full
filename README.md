# Tripleten web_project_api_full
This project is a full-stack application developed as a final sprint assignment, focused on implementing user authentication, authorization, error handling, and deployment. The application is divided into two main parts: a backend built with Node.js and Express, and a frontend developed with React. Users can register using an email and password, which are securely stored with hashing, and then log in to receive a JSON Web Token (JWT) used for accessing protected routes. The system includes functionality to retrieve current user data and ensures that users can only access and modify their own information. All protected routes require authorization, while registration and login remain public.
The backend also includes centralized error handling to manage unexpected issues and return appropriate HTTP status codes, as well as request validation using Celebrate/Joi and additional validation with the validator library. Logging is implemented to track both requests and errors. On the frontend, the JWT is stored locally and sent with API requests to maintain user sessions.
The project is structured into separate frontend and backend directories, and the application is deployed on a cloud server with Nginx configured as a reverse proxy. HTTPS is enabled to ensure secure communication, and environment variables such as the JWT secret are managed through a .env file on the server. Process management is handled with PM2 to ensure the application restarts automatically in case of failure. Overall, this project demonstrates a complete workflow of building, securing, and deploying a modern web application.
